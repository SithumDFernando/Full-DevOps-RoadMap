# Module 14: Capstone Projects

Congratulations on making it this far! You've learned about Linux, Networking, Git, CI/CD, Containers, Kubernetes, Helm, Terraform, Cloud, Monitoring, and Security.

Now it's time to put it all together. Reading about tools doesn't make you a DevOps Engineer; **building with them does.**

## 🎓 The Projects

We have designed three capstone projects, scaling in difficulty from Beginner to Advanced. Choose the one that best fits your current comfort level, or do all three!

1. [Project 1: The Static Website Pipeline (Beginner)](project-1-static-site.md)
2. [Project 2: The Three-Tier Application (Intermediate)](project-2-three-tier-app.md)
3. [Project 3: GitOps with ArgoCD (Advanced)](project-3-gitops.md)

## 🛠️ General Rules for Capstones

- **No copy-pasting:** Try to write the YAML and scripts from memory or by referencing the official documentation, not previous modules.
- **Use source control:** Create a brand new Git repository for your capstone project. Commit often.
- **Automate everything:** If you click a button in a UI to deploy something, you fail. Everything must be coded.
- **Tear it down:** Don't forget to `terraform destroy` or `helm uninstall` when you are done to avoid cloud bills!

Good luck!
