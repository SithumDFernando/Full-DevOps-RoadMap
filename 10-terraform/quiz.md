# Quiz 10

1. What is the difference between declarative and imperative IaC?
<details><summary>Answer</summary>
Declarative (Terraform) focuses on the desired end state. Imperative (bash scripts) focuses on the step-by-step commands to achieve the state.
</details>

2. Why is it dangerous to commit `terraform.tfstate` to Git?
<details><summary>Answer</summary>
The state file stores everything about your infrastructure in plain text JSON, including sensitive information like initial database passwords or private keys generated by Terraform.
</details>

3. What does `terraform plan` do?
<details><summary>Answer</summary>
It compares your `.tf` code against the current state file and the real-world infrastructure, and outputs a summary of what it will add, change, or destroy if you run apply.
</details>

4. Why use remote state (like S3) instead of local state?
<details><summary>Answer</summary>
So multiple team members can share the same state file. It also provides a central, secure location for state storage rather than a developer's laptop.
</details>

5. What problem does State Locking (via DynamoDB) solve?
<details><summary>Answer</summary>
It prevents two CI pipelines or developers from running `terraform apply` at the exact same time, which could corrupt the state file or cause race conditions in the cloud provider.
</details>
